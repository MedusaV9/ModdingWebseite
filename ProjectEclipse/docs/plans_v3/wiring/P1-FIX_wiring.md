## Behavior and persistence

- New freezes and `refreeze stages|all` snapshot the currently reloaded
  `EclipseConfig.stages(...)` radius table, including previous-radius inheritance for sparse
  stages. `refreeze stages|all` re-reads global `stages.json` first.
- Terrain, cave, map-wobble and ore fields in the owned files are keyed by
  `FrozenParams.mapSeed()`. Simplex caches carry their seed and rebuild on a same-JVM save
  switch. Terrain/ore hashes read the active frozen seed directly.
- `disc_heightmap.png` is copied byte-for-byte into `world/<save>/eclipse/` on first freeze
  and explicit map/all refreeze; frozen map loading reads only that save-local copy.
- Ancient debris defaults to unlock stage 2.
- Pending and placed structure rows now share
  `EclipseWorldgenState.pendingStructures()` with a `status` discriminator. A successful
  one-time read migrates and removes legacy `eclipse/pending_structures.json`.
- `SitePrep` plateau/cavity work is cursor-based and round-robin queued under the configured
  block-writer millisecond budget. Async registry rows remain pending until their completion
  callback.
- Mineshaft locate queries use deterministic `UndergroundSites` rows through the committed
  stage instead of looking for a nonexistent authored landmark.
- Ancient City remains anchored at y `-40`; the mountain deep-dark ceiling is raised to
  exclusive y `-32`, so the anchor lies in Deep Dark as D6/D7 specifies.
- Fog sites enqueue `eclipse:fog_storm` pending rows, use async `SitePrep`, and persist chest
  positions plus `placed`/`active` flags. `FogStormSites` restores active rows on boot;
  `stormfx.StormRegistry.pollFogSites()` then recreates the standing walls.

## In-game verification

1. Fresh freeze:
   - Set `randomizeMapSeed=true`; create two fresh saves.
   - Run `/eclipse-worldgen freeze status` in each and confirm different seeds and visibly
     different base terrain/ore/cave layouts.
   - Restart each save and confirm its seed and sampled terrain are unchanged.
2. Stage freeze:
   - Edit global `stages.json`; restart an existing save and confirm its freeze status/radii
     remain unchanged.
   - Run `/eclipse-worldgen refreeze stages`; confirm the status now shows the edited radii.
3. Heightmap:
   - Create a fresh save with a global `disc_heightmap.png`, confirm the save-local PNG exists,
     then alter/remove the global PNG and restart; terrain must retain the frozen image.
4. Ore:
   - In a nether stage-1 annulus confirm ancient debris is absent; at stage 2 confirm it can
     occur. Existing chunks do not reroll after config reload/refreeze.
5. Structures:
   - Commit a structure-bearing stage, run `/eclipse-worldgen structures list`, restart while
     rows are pending, and confirm the same rows resume.
   - Trigger one with `/eclipse-worldgen structures place <siteId>`; confirm the plateau/cavity
     advances over ticks and the row moves to placed only after completion.
   - Run `/locate structure minecraft:mineshaft` after overworld stage 2.
   - Locate/place the Ancient City and verify F3 reports `minecraft:deep_dark` around y `-40`.
6. Fog:
   - Commit overworld stage 3; confirm both authored fog landmark ids become pending and then
     materialize with three chests each.
   - Restart and confirm the sites remain active, chest positions remain indexed, and
     `StormRegistry` recreates both walls.
   - Erase below stage 3 and confirm the walls retire and state clears.
7. Textures:
   - Enter the Pale Garden and inspect log ends/sides, stripped logs, planks, leaves, moss
     block/carpet, and hanging moss in-world and in inventory; no missing-texture checkerboards.

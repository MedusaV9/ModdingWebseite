# P1-W1.7 wiring — Nether breach

## Files

- `network/S2CBreachPayload.java` — `{phase, center, radius}`, phases `QUAKE`, `OPEN`, `SETTLED`.
- `network/breach/BreachPayloads.java` — self-registering MOD-bus payload registrar and P2 consumer hook.
- `worldgen/nether/BreachBuilder.java` — Nether-stage-1 listener; budgeted arrival chimney, live crater/creep sweep, return pad, persistent `breachOpen` commit; public dev seam `openNow(ServerLevel)`.
- `worldgen/nether/BreachTransferService.java` — 8:1 mapped player descent, slow-fall arrival, soul updraft return, portal cancellation, pearl/elytra/hard-border clamps, baseline smoke/ash/sound.
- `worldgen/nether/HangingCourtBuilder.java` — Nether-stage-2 pending sites for the budgeted Hanging Court and vanilla bastion remnant.
- `worldgen/nether/NetherUndersideDecor.java` — self-registers through `DiscGenPipeline.registerExtraDecor`; deterministic glowstone, vines/roots, shroomlight, ash crust and below-disc ash ambience.
- `data/eclipse/loot_table/nether/hanging_court.json` — Court chest loot.
- `docs/plans_v3/langdrop/P1-W1.7.json` — intentionally empty: this package emits no user-facing text.

## Orchestrator hooks

No `EclipseMod.java` line is required. Every game handler is an `@EventBusSubscriber`, underside decor registers from its own MOD-bus common-setup subscriber, and `BreachPayloads` owns its `RegisterPayloadHandlersEvent` registration.

Do **not** also register `S2CBreachPayload` in `network/EclipsePayloads.java`; duplicate payload registration throws during startup.

P5's `/eclipse-worldgen breach open` branch should call:

```java
BreachBuilder.openNow(server.overworld());
```

`openNow` deliberately repairs/replays even when `EclipseWorldgenState.breachOpen()` is already true. The normal stage listener only starts it when the persisted flag is false.

No `BreachGeometry.java` change is requested. W1.7 materializes its existing `carveAt`/`creepAt` output directly.

## P2 FX ask

Install a dedicated-client consumer with:

```java
BreachPayloads.setClientPhaseHandler(payload -> { ... });
```

- `QUAKE`: begin camera shake, dust fall and low-frequency rumble.
- `OPEN`: start crater smoke/embers and the downward tear while the live block sweep advances.
- `SETTLED`: decay quake, keep a low persistent smoke/ash column.
- `center` is the overworld lip-plane block position; `radius` is the 16-block crater mouth radius.

The server already emits conservative campfire smoke, ash, soul flame and vanilla sounds, so the event remains legible without P2.

## Manual test steps

1. Use a fresh world, complete intro fusion, then commit Nether stage 1 (day 2) or call `/eclipse-worldgen breach open`.
2. Observe `QUAKE → OPEN → SETTLED`; verify the crater at `eclipse:nether_breach` has a pierced bedrock lens and crimson/netherrack/fire halo.
3. Fall through the central spout. Verify arrival at `eclipse:breach_arrival`, 8 seconds of Slow Falling, and no fall damage.
4. Enter the soul updraft and ride it to the top. Verify return above the inward overworld rim pad with brief Slow Falling.
5. Try lighting an obsidian portal (must fail), pearling through the deep funnel wall (target is pulled into the spout), and elytra/pearl travel beyond Nether stage radius + 16 (must clamp).
6. Commit Nether stage 2. Verify the top bastion pending site and the under-disc Hanging Court; descend either ladder shaft and open both loot chests.
7. Stop the server during crater/Court writes, restart, and verify deterministic completion resumes via the false breach flag / absent Court marker.
8. Re-run `/eclipse-worldgen breach open`; verify no duplicated block entities or structures and deterministic repair.

## Known scope

Only players cross dimensions. Riders are dismounted; boats, pets and other entities remain behind, matching D10's minimum required scope.

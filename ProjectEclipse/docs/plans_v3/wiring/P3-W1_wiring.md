## `registry/EclipseSounds.java` — 10 new `ui.*` events (Ledger)

`UiSounds` resolves these BY ID at runtime (`BuiltInRegistries.SOUND_EVENT.getOptional`)
and falls back to pitched existing events until they are registered, so **nothing breaks
if this lands late** — but register them so subtitles/volume come from `sounds.json`:

```java
public static final Supplier<SoundEvent> UI_CLICK = SOUNDS.register(
        "ui.click", () -> SoundEvent.createVariableRangeEvent(
                ResourceLocation.fromNamespaceAndPath(EclipseMod.MOD_ID, "ui.click")));
```

…same shape for: `UI_TOGGLE` (`ui.toggle`), `UI_SLIDER` (`ui.slider`), `UI_ERROR_GLITCH`
(`ui.error_glitch`), `UI_LEVEL_UP` (`ui.level_up`), `UI_SKILL_BUY` (`ui.skill_buy`),
`UI_ROULETTE_TICK` (`ui.roulette_tick`), `UI_ROULETTE_WIN` (`ui.roulette_win`),
`UI_TIMER_ZERO` (`ui.timer_zero`), `UI_GHOST_BURST` (`ui.ghost_burst`).

## Notes per sibling

- **W1.3 (ores) + W1.9 (freeze) — `annulusBand` nether gap found while scan-testing**:
  `OreField.tryOre` gates every ore by `FrozenParams.annulusBand(cellR)`, and the §3.10
  seam `annulusBand(double r)` has no profile parameter. If W1.9 implements it on the
  overworld radii (as §3.10/D9 read), NETHER cell radii (r ≤ ~170 < 225) always land in
  band 0-1, while all three default nether ores (quartz/nether_gold/netherite) have
  `unlockStage` 2-3 → nether ores can never generate. Verified empirically: a 800k-state
  nether scan with overworld-band semantics yields 0 ores; forcing band 5 yields
  quartz/gold/debris as configured. Either `annulusBand` should resolve bands against the
  radii set of the disc the radius belongs to (nether radii {64,110,150} — but the two
  discs' radius ranges overlap, so better: add the profile to the call), or OreField
  should map nether stages itself. Flagging for integration — not fixable inside W1.2.
- **W1.1 (vanilla pipeline)**: the terrain function no longer emits ANY vegetation —
  no trees, grass/flowers, dead bushes, cacti, snow LAYERS, nether roots/vines/fungi,
  bone/basalt pillars, soul fire. All of it must come from real biome features
  (`freeze_top_layer` included) — do not deny-list vegetation/snow features. Cave band
  now reaches the surface (`caveMaxY == surfaceY`) so worm mouths may daylight; cheese
  rooms fade near rim/underside on their own. `CaveEntrances` craters expect glow-lichen
  rims from decoration (D4.4). `DiscColumn` record components changed (removed
  cover/pillar/tree, added swampPool/caveFade/entrance/breach/endDisc — the accessors
  you use, `inside/bottomY/topY/surfaceY/lavaTopY`, are untouched). After the End disc
  materializes, `topY`/heightmaps of columns under it jump to ~y 360-410; W1.4 maps
  y>320 there to `minecraft:the_end`, so decoration ground probes stay sane.
- **W1.4 (biome map v2)**: please author the landmark `eclipse:nether_breach` in
  `DiscMapDefaults` — `BreachGeometry` anchors on it and falls back to (x 92, z 92)
  (r≈130, desert wedge mid-angle 45°) when absent. Note the crater (r 16 + 12 halo)
  visually clips the stage-1 rim (r 150); consider r≈120-130 with that in mind.
  `styleOf` already palettes the whole Appendix-A biome table (badlands→terracotta
  mesas, mushroom_fields→mycelium, pale_garden→dark-forest grass, etc.); unknown ids
  still fall back to plains/nether_wastes. Desert dunes orient by the CONTAINING
  desert/badlands wedge's mid-angle from the live `DiscMapData` sector list — ring
  sub-splits keep the wedge angles, so nothing to do. Cave/end biomes: use the stable
  `DiscTerrainFunction.surfaceY` and `EndDiscGeometry.footprintContains`/`MIN_Y`.
- **W1.7 (breach builder/transfer)**: simplest materialization = flip
  `EclipseWorldgenState.breachOpen` (mirrored in FrozenParams), then rewrite all columns
  with `BreachGeometry.contains(x, z)` via `DiscTerrainFunction.column`/`stateInColumn`

  (`carveAt`/`creepAt` remain available for per-block use.) The funnel pierces the

  fall into the void — your `BreachTransferService` catch zone is the chimney column at

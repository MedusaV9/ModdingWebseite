# P1-W1.3 — Ore gate engine wiring

Owner: `worldgen/ore/*`. Integrator / sibling workers apply these hooks; **do not** edit
`EclipseMod.java`, `EclipseCommands.java`, or lang JSONs from this package.

## Required sibling: `FrozenParams` (W1.9)

`OreField` and `OreGateApi` call `dev.projecteclipse.eclipse.worldgen.FrozenParams.annulusBand(double r)`
per §3.10. **W1.9 must land `FrozenParams.java`** before this package compiles. Expected signature:

```java
public static int annulusBand(double r);
```

Implementation should derive band boundaries from **frozen per-save** stage radii
(`FrozenParams.stageRadii(DiscProfile)`), not hardcoded legacy constants.

## `EclipseConfig.reload()` (W1.9)

After config dir is resolved, call:

```java
OreConfig.reload(configDir); // same Path as other config/eclipse/*.json files
```

On first boot this writes `config/eclipse/ores.json` with day-gated defaults.

## Per-save freeze (W1.9 / D9)

Copy the loaded `ores.json` snapshot into `world/<save>/eclipse/worldgen.json` on first save
boot. On `ServerAboutToStartEvent`, reload ore rules from the **frozen** copy (either re-call
`OreConfig.reload` against a save-local directory containing `ores.json`, or inject a frozen
JSON section — W1.9 owns the freeze plumbing). Global `config/eclipse/ores.json` remains the
template for **new** saves only.

## `DiscTerrainFunction` (W1.2)

Replace private `oreAt(DiscColumn, int)` with:

```java
OreField.oreAt(col.profile(), col.x(), y, col.z(), y < col.deepslateTopY());
```

Remove `OreType`, `OVERWORLD_ORES`, `NETHER_ORES`, and local `annulusBand`.

## `/eclipse-worldgen ores reload` (P5 command class)

Thin wrapper:

```java
OreConfig.reload(FMLPaths.CONFIGDIR.get().resolve("eclipse"));
```

Already-generated chunks keep old blocks (documented behaviour).

## Lang merge

Merge `docs/plans_v3/langdrop/P1-W1.3.json` → `assets/eclipse/lang/en_us.json` and `de_de.json`.

## P4 `OreGateApi` usage

- `unlockStageOf(oreId)` — goal/UI text and ModGate day alignment with `ores.json`
- `unlockedInBand(band)` — list ores available when mining in that annulus
- `bandAt(profile, pos)` — annulus index at a world position (XZ distance from origin)

Note: `bandAt` accepts `DiscProfile` per §3.10 but band indexing is currently driven solely by
`FrozenParams.annulusBand(r)` until W1.9 adds profile-specific boundaries if needed.

## Notes for P6-W3 (ship rebuild — oar anchors + bench contract)

1. **The block-display oars are gone.** Do NOT place oar displays, oarlock display
   anchors, or reserve `EclipseWorldState.getOarEntities()` — the oar is bone
   `oar/oar_loom/oar_shaft/oar_blade` inside `deckhand.geo.json`. The persisted oar-UUID
   list is being drained to empty by `OarAnimator`'s migration and must stay empty.
2. **Keep calling** `DeckhandEntity.ensureCrew(limbo)` and `OarAnimator.ensureOars(limbo)`
   from `GhostShipBuilder.onServerStarted` (both signatures unchanged; `ensureOars` is
   now the legacy-display purge and is what cleans old worlds after your rebuild).
3. **Bench geometry consumed by W2** (must stay true in ship v2, or the crew seats
   wrong): benches at x ∈ {−12,−4,4,12} on both sides, seat block =
   `deckY = GhostShipBuilder.waterlineY(limbo) + 3`, seat Z =
   `side * (GhostShipBuilder.halfWidthAt(x) − 1.0)` — i.e. exactly one block inboard of
   the gunwale, rower facing outboard. W2 reads `waterlineY` + `halfWidthAt` live, so
   hull-curve changes propagate automatically; if v2 moves the deck off
   `waterline + 3` or re-columns the benches, ping W2 (constants `BENCH_X` and the
   `+3` in `DeckhandEntity.benchCenter`).
4. **After a v2 rebuild**, previously persisted rowers still sit at their OLD seat
   coordinates. Cheap fix already in the API: call `DeckhandEntity.calmCrew(limbo)`
   once after the rebuild — it is idempotent and snaps every living crew member onto
   its (new) bench. (`reseatFallen` only replaces dead/missing rowers; `calmCrew`
   repositions living ones.)
5. **Gunwale/oarlock cosmetics:** if v2 carves oarlock notches, align them with the
   bench X columns. The model's oar exits the hull ~0.9 blocks above the seat and
   reaches ~2.5 blocks outboard; at the catch the blade tip dips ~1.1 blocks below
   deck level. The renderer finds the actual water surface by probing up to 7 blocks
   down from seat height for the splash — a freeboard (deck − waterline) of up to ~7
   blocks is tolerated without code changes; taller and the probe depth in
   `DeckhandRenderer.spawnCatchSplash` needs a bump.

## Behavior contract (for the orchestrator's QA pass)

- Boot, fresh world → 8 rowers seated, all stroking in unison (shared 60 t clock),
  blades feathering on the return, splash on the catch; port and starboard blades both
  drive stern-ward.
- `/summon eclipse:ferryman` shortly after boot → `reseat window opened` log, NO
  duplicate rowers even if entity sections are still loading (4a).
- Kill the summoner mid-P2 (crew risen) → wipe/reset path runs `calmCrew` → every
  surviving deckhand snaps back to its bench, rows again, and is invulnerable-passive
  (4b). `/kill @e[type=eclipse:ferryman]` mid-P2 → within ~5–6 s the crew self-calms
  and reseats (orphan self-heal).
- Old worlds: `Ghost ship oars: discarded N legacy block-display oar(s)` (+ possible
  positional-sweep line) on first boot; afterwards `no legacy block-display oars left`.

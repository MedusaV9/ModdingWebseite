## 1. `EclipseMod` (orchestrator) — REQUIRED before boot

Same pattern/place as the existing classic-blocks register call. Required before any

## 2. `DiscMapData` (W1.9's file) — v2 defaults + ring resolution (seam marked there)

`DiscMapData.fromJsonRoot` javadoc already marks this seam ("or DiscMapDefaults when
W1.4 lands"). Two surgical patches:

**(a) Built-in defaults → authored v2 defaults** (activates the v2 landmark table:
mansion / outpost / trial chambers / ancient city / nether breach / breach arrival /
bastion / hanging court / fog storms / stronghold-at-rim — plus center cap
`minecraft:plains` per Appendix A):

```java
private static MapProfile defaultOverworld() {
    return DiscMapDefaults.overworldDefaults();
}

private static MapProfile defaultNether() {
    return DiscMapDefaults.netherDefaults();
}
```

(The now-dead inline default bodies can be deleted; `DiscMapDefaults` keeps the v1
rivers/wells byte-identical, so `BreachGeometry` lip planes and W1.2 spill curtains are
unaffected.)

**(b) Sub-ring resolution in the shared `biomeAt` path** — W1.2's
`DiscTerrainFunction.styleOf` already covers every ring id (badlands strata,
pale_garden→DARK_FOREST palette, …) and its javadoc says it activates "the moment the
new map defaults ship". Wrap only the **wedge** return of `DiscMapData.biomeAt` (last
line — mountain core/flank and center cap stay un-ringed):

```java
return DiscMapDefaults.ringBiome(profile,
        sectorBiomeAt(map, wobbledAngleDeg(map, x, z, r)), x, z);
```

`ringBiome` is pure, thread-safe and **idempotent** (ring outputs pass through
unchanged), so `DiscBiomeSource` applying it a second time on the same position is
harmless-by-design. Without patch (b) the game still works, but mid/outer-ring columns
keep the wedge base palette (e.g. badlands ring generates desert sand strata while the
biome reads `minecraft:badlands`).

## 3. `FrozenParams` (W1.9's file) — pre-existing compile error (blocks compileJava)

`createInitialFreeze` reassigns `mapSeed` then captures it in the
`defaultFogstormsJson(mapSeed)` lambda → "local variables referenced from a lambda
expression must be final or effectively final". One-liner:

```java
final long frozenSeed = mapSeed;   // before root.add("fogstorms", ...)
... () -> defaultFogstormsJson(frozenSeed)));
```

W1.4's sandbox verification shadow-fixed this locally; the repo file still carries it.

## 6. Fog storms — W1.9 coordination (single source of truth)

W1.9's `FrozenParams.defaultFogstormsJson` seed-picks 2 sites at mid-stage-3 radius.
The authored `eclipse:fog_storm_1/2` landmarks sit at exactly that radius
(r ≈ 245 / 250, angles 225° / 270°), placed to clear rivers/sites. Recommended:
`defaultFogstormsJson` should prefer the `fog_storm*` landmarks from
`DiscMapDefaults.overworldDefaults()` when present and only fall back to
`pickStormAngles` — otherwise saves get two authorities for storm positions (the
landmark clearance zones then protect the wrong spots). W1.4 deliberately did NOT touch
`FrozenParams`; until aligned, treat `fogstorms.json` as authoritative and the two
landmark rows as reserved-clearance markers only.

## 9. Misc integration notes

- **`DiscBiomeSource` construction order**: it resolves holders via `DiscMapData.get()`
  inside the codec/constructor path (unchanged from v1); the frozen `discMap` install
  (W1.9, `ServerAboutToStartEvent`) already runs before dimension load, so the per-thread
  column cache keying on the `DiscMapData` instance self-invalidates on install/reload.
- **End biome overlay is always-on** above y 320 inside `EndDiscGeometry.footprintContains`
  (biome pre-exists the D10 materialization flag harmlessly — plan-sanctioned).
- **Radial ring seams** get the ±6-block wobble but (unlike wedge seams) no relief blend;
  style pairs across ring boundaries have close offsets except desert→badlands, which
  reads as natural terracing. If W1.2 ever wants radial blending, hook it next to
  `sectorBlendAt`.
- **Datapack**: all pale-garden JSONs ship in the mod's built-in datapack; no
  `pack.mcmeta`/registration needed. `/locate biome eclipse:pale_garden` works once the
  map defaults patch (§2a) is applied (outer plains wedge, r ≳ 300).

# P5-W8 wiring notes (classic blocks)

W8 ships the complete `eclipse:classic_*` set self-contained under
`classicblocks/**`, `tools/classicblocks/**`, generated resources and bundled
textures. Exactly **one** foreign line is needed.

## EclipseMod wiring (integrator)

In the `EclipseMod` constructor, next to the other registry hookups:

```java
dev.projecteclipse.eclipse.classicblocks.ClassicBlocksModule.register(modEventBus);
```

That registers all three DeferredRegisters (188 blocks, 172 items, the
`eclipse.classic` creative tab). Nothing else: no config, no payloads, no
client-setup — cutout/translucent render layers are baked into the model JSONs
via the NeoForge `render_type` field (157 models carry it), so do NOT add
`ItemBlockRenderTypes` calls.

## Langdrop

`docs/plans_v3/langdrop/P5-W8.json` — **189 keys per language**:
188 × `block.eclipse.classic_<id>` (en `Classic — <Block>`, de
`Klassisch — <Block>`) + `itemGroup.eclipse.classic`. Merge into the src lang
JSONs as usual. Names are generated from the manifest
(`tools/classicblocks/manifest.py`); re-running
`tools/classicblocks/gen_assets.py` rewrites the langdrop deterministically.

## For P5-W9 (event runtime)

* **Chest loot hook** (plan §2.14 "Chest loot"): during common setup call

  ```java
  ClassicChestLoot.setProvider(XboxEventService::lootFor);
  ```

  Breaking a `classic_chest`/`classic_trapped_chest` calls
  `ClassicChestLoot.lootFor(serverLevel, pos)` and spills the returned stacks
  **in addition to** dropping the chest block itself. Unset provider (or an
  empty list) = the chest just drops itself — safe outside event dims.
* **Obtainability**: classic items are event loot only. They sit in the
  dedicated `eclipse.classic` creative tab (ops/testing), never in vanilla
  tabs, and have zero recipes (separate namespace, no vanilla/common item
  tags ⇒ they satisfy no ingredient).
* Registry access for loot mapping: `ClassicBlocks.byId("<path-without-prefix>")`
  / `ClassicBlockItems.byId(...)` (e.g. `byId("oak_planks")` →
  `eclipse:classic_oak_planks`).

## Deviations (sanctioned)

* **Python generator instead of gradle `runData`**: worker instructions
  override the plan's datagen bullet — assets are generated by
  `tools/classicblocks/gen_assets.py` into `src/generated/resources`
  (939 JSONs), no build.gradle change, no datagen run configs.
* `Blocks.SNOW`-parity `forceSolidOff()` on `classic_snow` emits a NeoForge
  deprecation *warning* at compile time (vanilla itself uses the same call;
  no `-Werror` in the build).

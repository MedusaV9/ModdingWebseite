# P1-W1.8 wiring — End disc and dragon fight

## Files

- `worldgen/end/EndConfig.java` — save-local `end.json` snapshot and `ReloadHooks` registration.
- `worldgen/end/EndFightState.java` — restart-safe materialization cursor, crystal UUID/count, dragon identity/position/health, death timestamp, and victory flag.
- `worldgen/end/EndDiscService.java` — day/stage/flag trigger polling, crash announcement, budgeted terrain + mini-city materialization, restart resume, and fight startup.
- `worldgen/end/EndSpires.java` — crystal reconciliation and the deterministic upper/hanging purpur mini-cities.
- `worldgen/end/EclipseDragonFight.java` — overworld dragon lifecycle, fight-origin/phase watchdog, crystal healing and kill gate, boss bar, centered death, rewards, and Sanctum-return portal.
- `network/end/S2CEndCrashPayload.java` — disc center/radius plus client timeline and shake hints.
- `network/end/EndPayloads.java` — self-registering MOD-bus payload registrar and P2 consumer hook.
- `data/eclipse/loot_table/end_city/cache.json` — default mini-city cache; intentionally contains no elytra.
- `data/eclipse/loot_table/end_city/cache_with_elytra.json` — opt-in cache selected only when `allowElytra=true`.
- `docs/plans_v3/langdrop/P1-W1.8.json` — en_us/de_de boss and announcement keys.

The required world-height work is already present: `DiscProfile.OVERWORLD` and
`data/minecraft/dimension_type/overworld.json` both use `min_y=-176`, `height=640`
(top build Y 463), and `logical_height=640`. `EndDiscGeometry` already contains the
flag-gated lens, podium, eight spires, two cages, and chorus fields; no read-only
geometry change is requested.

## Orchestrator hooks

No `EclipseMod.java` line is required. `EndConfig`, `EndDiscService`, and
`EclipseDragonFight` are game-bus `@EventBusSubscriber`s; `EndPayloads` owns its
`RegisterPayloadHandlersEvent` registration on the MOD bus.

Do **not** also register `S2CEndCrashPayload` in `network/EclipsePayloads.java`;
duplicate payload registration throws during startup.

The existing `/eclipse-worldgen end materialize` command only needs to set
`EclipseWorldgenState.endDiscMaterialized=true`. `EndDiscService` polls that flag
every 20 ticks and starts or resumes the writer. Direct callers may instead use:

```java
EndDiscService.materialize(server);
```

`EndConfig` registers the `end` key with `ReloadHooks`, so the shared
`/eclipse reload` path needs no direct call. If the separate
`/eclipse-worldgen reload` command is expected to reload this section too, its owner
should add `EndConfig.reload(saveDir)` beside the ore/fog reload calls.

P4 or an analytics bridge can subscribe once to the victory seam:

```java
EclipseDragonFight.addListener((server, center) -> { ... });
```

`AnalyticsApi` is read-only, so W1.8 does not fabricate an analytics write path.

## P2 FX ask

Install a dedicated-client consumer during client setup:

```java
EndPayloads.setClientCrashHandler(payload -> { ... });
```

- Start the sequence at `payload.center()` and size its impact/rift to `payload.radius()`.
- Use `timelineTicks` (server default 120) for sky-tear, falling debris, impact flash,
  shockwave, and decay timing.
- Use `shakeStrength` as the maximum camera impulse. The server also sends the shared
  `S2CShakePayload` and plays dragon-growl + thunder cues, so the arrival remains legible
  without the dedicated effect.
- Optionally subscribe to `EclipseDragonFight.Listener` server-side for victory sky/fog
  cleanup; the server already emits the portal sound and a shorter shake.

## Config

The frozen `end.json` contract is:

```json
{
  "trigger": "day:9",
  "centerX": 0,
  "centerZ": 0,
  "surfaceY": 360,
  "radius": 96,
  "crystalCount": 8,
  "dragonHealth": 300.0,
  "simpleDragonAi": true,
  "allowElytra": false,
  "crystalRespawn": false,
  "lootTable": "eclipse:end_city/cache",
  "elytraLootTable": "eclipse:end_city/cache_with_elytra",
  "victoryXp": 12000,
  "blockBudgetPerTick": 768
}
```

Geometry values are validated but fixed to `DiscProfile`/`EndDiscGeometry`; accepting
different live-writer geometry would create seams with chunks generated after arrival.
Valid trigger forms are `day:N`, `stage:N`, and `final_day`.

## Manual test steps

1. Use a fresh event world. Set `end.json` before freeze or run
   `/eclipse-worldgen end materialize`.
2. Confirm the arrival announcement, growl/thunder, shared camera shake, and P2 crash
   payload. Verify materialization progresses without one-tick bulk placement.
3. Teleport above `(0, 360, 0)`. Verify an End-biome end-stone lens, eight obsidian
   spires, two iron-bar cages, eight crystals, podium, and chorus fields.
4. Inspect the purpur tower at approximately `(-68, 360, 18)` and the hanging room at
   `(58, 333, -50)`. Open both caches and confirm the default table has no elytra.
5. Stop during terrain or mini-city writes, restart, and verify the saved cursor resumes
   without duplicate entities or structures.
6. Verify the purple boss bar appears within 192 blocks. Damage the dragon while a
   crystal survives: it must heal and cannot be reduced below one health.
7. Break all crystals. Verify the announcement, landing/perch attempts, and that lethal
   damage is now accepted. Check that an out-of-bounds dragon is returned to the disc.
8. Restart once during the live fight and once during the death animation. Verify UUID
   reattachment (or safe reconstruction), preserved health/crystal count, and one victory.
9. After the centered death sequence, verify the XP shower, egg, lit podium portal, and
   victory announcement. Enter the portal and confirm it returns the player to the
   Overworld Sanctum spawn instead of loading the vanilla End or credits.
10. Re-run the materialize command after completion and restart again. Verify no second
    dragon, crystal set, reward shower, egg, or mini-city is created.

## Integration risks

- Vanilla dragon phases were designed around the End. `setFightOrigin` is used on every
  control tick; the watchdog bounds the entity and `simpleDragonAi=true` limits the
  crystal phase to holding/strafe until the crystals are gone.
- The eight pillar chunks and the dragon chunk receive renewable short-lived writer
  tickets while the fight is active so unloaded crystals are not mistaken for destroyed
  crystals.
- Real End portal blocks are retained for the visual. `EntityTravelToDimensionEvent`
  cancels only managed, victorious podium travel and redirects players to the shared
  Overworld spawn, which the Sanctum builder pins to its south approach/plaza.

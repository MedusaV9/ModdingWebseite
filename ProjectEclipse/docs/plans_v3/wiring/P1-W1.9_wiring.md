## Per-save layout

| Path | Content |
|---|---|
| `world/<save>/eclipse/worldgen.json` | Master freeze (mapSeed, stageRadii, discMap, ores, end, fogstorms) |
| `world/<save>/eclipse/ores.json` | Extracted ore snapshot (OreConfig.reload target) |
| `world/<save>/eclipse/fogstorms.json` | Extracted fog sites |
| `world/<save>/eclipse/end.json` | Extracted end disc params (until W1.8 EndConfig owns load) |
| `world/<save>/eclipse/fog_storm_loot.json` | Per-site chest loot table ids |

Global `config/eclipse/*` remains the template for **new** saves + live-reload runtime knobs
(`worldgen_tuning.json` stays global-only per plan).

## D8 radii defaults (fresh saves)

- Overworld: `{96, 150, 210, 280, 360, 440}`
- Nether: `{0, 64, 110, 150}`
- Legacy fallback when `worldgen.json` absent **and** `EclipseWorldState` has committed stages:
  `{96, 225, 300, 360, 420, 480}` / `{0, 80, 120, 160}`

`EclipseConfig.defaultStages()` and `StageRadii` initial statics updated to D8 values.

## `annulusBand` nether fix (W1.2 gap)

`OreField.tryOre` now calls `FrozenParams.annulusBand(profile, cellR)` so nether ores resolve
bands against `{0, 64, 110, 150}` instead of overworld radii. `OreGateApi.bandAt` updated
similarly. The §3.10 `annulusBand(double)` overload remains overworld-biased for compat.

## Materialization flag mirrors

`FrozenParams.breachOpen()` / `endDiscMaterialized()` read volatile mirrors on the frozen
context. **W1.7 / W1.8** must call `FrozenParams.mirrorMaterializationFlags(...)` when flipping
`EclipseWorldgenState` flags (W1.5). Until then mirrors stay false on new chunks.

## DiscMapData load order (D9)

1. Frozen `discMap` section from `worldgen.json` (via `DiscMapData.install`)
2. Global `config/eclipse/disc_map.json` (when no active freeze)
3. Built-in defaults (`defaultOverworld()` / `defaultNether()`) — W1.4 `DiscMapDefaults`
   should replace step 3 when it lands; `fromJsonRoot` javadoc marks the seam.

`DiscMapData.clearInstance()` on `ServerStoppedEvent` fixes the JVM-global singleton bug.

## Fog storms

- Default two sites auto-picked in stage-3 annulus (mid-r between radii[2] and radii[3])
- `FogStormSites` listens on `WorldStageService` stage terrain complete (stage 3)
- Places grove + 3 chests + broadcasts `S2CFogStormPayload`
- Loot: `data/eclipse/loot_table/fog_storm/storm_cache.json` (Mending + `eclipse:replant` hook)

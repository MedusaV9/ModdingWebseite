The P1 plan's orchestrator-wiring section lists `S2CGrowthWavePayload` for

`eclipse:growth/wave`). Nothing to merge — the registrar is live via `@EventBusSubscriber`.

## Frozen seam for P2-W7 — growth pacing hooks (design D11)

`worldgen.stage.GrowthPacing` (thread-safe volatile snapshot reads; all static):

```java
public static int targetTicks();        // animated sweep paces toward this duration (default 1500)
public static int revealDelayTicks();   // chunk resend lag behind covering wavefront pulse (default 10)
public static int ringsPerPulse();      // max 1-block rings the front advances per 5-tick pulse (default 5)
public static int shakeEveryRings();    // S2CShakePayload cadence in rings; <= 0 disables (default 25)
```

```json
              "shakeEveryRings": 25, "columnRiseTicks": 30 },
```

## Frozen seam for P2 — wave payload + client hook

`S2CGrowthWavePayload {dim, fromStage, toStage, innerR, outerR, waveR, waveAngleStart,
waveAngleEnd, columnRiseTicks, pulseIndex}` (exact D11 fields; semantics javadoc'd on the
record) is broadcast every 5 ticks to players in the growing dimension during ANIMATED,
non-rebuild sweeps only. Guarantees:

- **Resend-after-wavefront**: a rewritten chunk's relight/resend happens ≥
  `growth.revealDelayTicks` after the pulse covering its columns went out — clients never
  see a raw chunk pop ahead of the animation front. Instant sweeps skip payloads, pacing
  and reveal delay entirely.
- `waveR` is the ring radius for GROW, the receded radius for ERASE, and the
  **distance-to-nearest-disc-edge** for the intro fusion (`fromStage == 0` overworld);
  fusion pulses always span the full `−π..π` angle range.
- `pulseIndex` restarts at 0 for resumed sweeps (restart-safe animation state).
- Shake pulses reuse the existing `S2CShakePayload` (`growth.shakeEveryRings` cadence),
  suppressed during the intro fusion (`FusionSequence` owns that rumble).

Client install (P2, from client setup — no edit to this package needed):

```java
GrowthPayloads.setClientWaveHandler(Consumer<S2CGrowthWavePayload> handler); // client main thread
```

Until installed, received pulses are debug-logged and dropped (server-safe class).

## Frozen seam for P4/P6 — `worldgen.stage.NewRingRegistry` (design D11)

```java

public static double freshness(ServerLevel level, BlockPos pos);       // 1 → 0 over glitch.freshTicks
```

- Rows are appended ONLY by genuine GROW stage commits (`WorldStageService.onSweepComplete`
  → package-private `onRingCommitted`, guarded `toStage > fromStage`); rebuilds and StageIO
  snapshot loads never flag, erase sweeps are filtered.
- `sampleFreshPositions` draws area-uniform positions in fresh annuli restricted to
  **currently loaded** chunks (never loads/generates a chunk); Y = motion-blocking surface
  + 1; may return fewer than requested.
- Server-thread only (reads SavedData rows).

## Behavioral notes for integrators

- **Sweep resume across restarts**: `onServerStopping` now drains up to 64 pending
  chunk-finishes synchronously and, if a backlog remains, rolls the persisted growth
  cursor back to the first column of the oldest unreplayed chunk — a resumed sweep never
  leaves a fully-written chunk without its decoration replay. Hard crashes can still lose
  replays since the last periodic cursor persist; the next commit's band overlap or
  `/eclipse stage rebuild` self-heals those chunks.
- **Entombment protection** (unchanged guarantee, now two-stage): base-rewrite rescue pops
  players out of freshly written columns; a second post-replay rescue pops players out of
  freshly decorated blocks (tree trunks) after each chunk's pipeline replay.
- `/eclipse reload` feedback for a post-freeze `features.deny` change: the filter logs a
  warning and keeps the frozen list until restart (see W1.1 wiring). Surface in command
  feedback if desired — P5's command class, not this package.

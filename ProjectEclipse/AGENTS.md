# ProjectEclipse — Cursor Cloud dev notes

**PROJECT: ECLIPSE** is a Minecraft **1.21.1 / NeoForge 21.1.238** server-event mod ("Eclipse-Core", mod id `eclipse`), an anonymous hardcore-survival psychothriller. This Gradle project lives in the `ProjectEclipse/` subdirectory of the repo (the repo root also contains an unrelated website — do not mix them). See `README.md` here for the full architecture, command list, 14-day plan, config files, and server-pack list.

## Cursor Cloud specific instructions

- **Java**: build/run with **Java 21** (installed system-wide). There is no system Gradle — always use the wrapper `./gradlew` from inside `ProjectEclipse/`.
- **Build / run** (standard, documented in `README.md`): `./gradlew build`, `./gradlew runServer` (auto `--nogui`; write `run/eula.txt` `eula=true` first), `./gradlew runClient`. The **first** `./gradlew build` downloads NeoForge + the MC toolchain and can take 10–30 minutes — allow very long timeouts and do not abort it. Subsequent builds are seconds.
- **Client runs on the VM desktop**: `runClient` opens a real Minecraft window that can be driven/inspected via the desktop (computerUse). OpenAL audio has no device here (harmless warning). This is how the title screen, purple sky, artifact menu, etc. are visually verified.
- **v2 REQUIRES A FRESH WORLD**: the overworld dimension type is now `min_y -176 / height 512` (floating-disc world) — v1 saves fail to load. Delete `run/world` once after upgrading; the disc/limbo dimensions regenerate deterministically.
- **Mods folder layout** (all gitignored): `run/mods/` holds the full server-safe v2 matrix (Create 6.0.10, Aeronautics-bundled 1.3.0, Sable 2.0.3, Voice Chat 2.6.16, Farmer's Delight 1.3.2, Supplementaries 3.8.3 + Moonlight 3.1.1, Sophisticated Backpacks 3.25.71 + Core 1.4.77, CreateAddition 1.6.0 — exact jars in README "Server pack") and is loaded by BOTH `runServer` and `runClient`. `run/mods-client/` holds Sodium 0.8.12 + Iris 1.8.14-beta.1 — CLIENT-ONLY: they crash the dedicated server in early bootstrap (`NoClassDefFoundError: org/lwjgl/Version`), so copy them into `run/mods/` only for client runs and remove before server runs. `run/mods-disabled/` parks the old Sodium 0.6.13/Iris 1.8.12 pack, which Veil 4.3.0 REJECTS (it enforces sodium `[0.8.12-alpha.2+mc1.21.1,)` and is incompatible with Embeddium) — do not resurrect it.
- **Sodium + Iris (client target)**: the tested client runtime is `sodium-neoforge-0.8.12+mc1.21.1` + `iris-neoforge-1.8.14-beta.1+mc1.21.1`. The custom purple-sun sky and all Veil post pipelines self-disable (return to Iris) when a shaderpack is active, detected via Veil's first-party `foundry.veil.api.compat.IrisCompat` wrapped by `client/sky/EclipseIrisState` (boot log line: `Iris detected (via Veil IrisCompat)`); without a shaderpack the full custom sky + Veil FX render. Veil 4.3.0 is a REQUIRED runtime dependency (jar-in-jar'd from `maven.blamejared.com`).
- **RCON for headless server smoke-tests**: `run/server.properties` ships `enable-rcon=true`, `rcon.port=25575`, `rcon.password=eclipsedev`. Piping into `runServer` stdin does NOT work under Gradle — use RCON (no `mcrcon` binary installed; a ~40-line Python socket client suffices) to drive `/eclipse status`, `day set`, `stage set`, boss summon/kill, etc., then kill the server by specific PID.
- **Voice chat / gated mods are soft/optional deps** (see README "Server pack"): all external mods are installed alongside Eclipse-Core, NOT jar-in-jar (their assets are All-Rights-Reserved, and jar-in-jar mods cannot be disabled at the loader level). Progression gating is server-authoritative by namespace (`ModGate`), so the build has ZERO compile-time dependency on them. The Aeronautics bundle also ships Create: Offroad (`offroad` namespace, gated under the `aeronautics` key). Aeronautics 1.3.0 pins Create `[6.0.9,)` — don't downgrade Create below 6.0.10.
- **Stopping lingering game instances**: `runServer`/`runClient` leave a `net.minecraft...devlaunch.Main` JVM running that holds the world lock / GL context and RAM (~1–3 GB each). Stop them by **specific PID** (`ps -eo pid,args | grep devlaunch.Main`, then `kill <pid>`) — never blanket-kill. Gradle daemons are reusable and can stay.
- **All UI text is de_de + en_us** in `assets/eclipse/lang/`. The mod is intended to be MANDATORY on clients (anonymity + anti-cheat rely on it).
- **Textures**: the large "hero" assets (title background/logo/panorama, purple sun, eclipse disc, artifact menu panel) are final art; the small pixel-art item/block icons are simple programmer-art placeholders at documented paths under `src/main/resources/assets/eclipse/textures/` — drop-in replaceable byte-for-byte.
- **No text-based tests**: verification is `./gradlew build` (compiles, strict) + `runServer` clean boot + `runClient` manual/visual checks. There is no unit-test suite.

# ModdingWebseite тАФ BAPBAP Modding

A multi-page marketing/community website for **BAPBAP Modding** (a fan-made community modding project for the Steam roguelike party game *BAPBAP*). Built with **Vite + React + TypeScript + Tailwind CSS v4** as a static SPA. Primary community CTA is the Discord `discord.gg/BAPBAPMods`.

## Cursor Cloud specific instructions

- **Node**: use the nvm-provided Node (v22.x). If `node`/`npm` are not on `PATH` in a fresh shell, run `source ~/.nvm/nvm.sh` first. The package manager is **npm** (there is a `package-lock.json`; do not use yarn/pnpm).
- **Run the app (dev)**: `npm run dev` serves on `http://localhost:5173`. There is no backend тАФ it's a purely static frontend, so no DB/env vars/services are required.
- **Other scripts**: `npm run build` (`tsc -b` in strict mode + `vite build`), `npm run preview` (serves the production build on `http://localhost:4173`).
- **Lint gotcha**: `npm run lint` runs **oxlint**, NOT eslint (this is what the current `create-vite` react-ts template ships). There is a `.oxlintrc.json`, no `eslint.config.js`.
- **Routing**: the site is multi-page via react-router-dom v7 **HashRouter** (deliberate тАФ the static host has no SPA fallback, so only `/#/...` deep links survive refresh). Routes live in `src/App.tsx`, page components in `src/pages/`, and the shared Navbar/Footer shell in `src/Layout.tsx`.
- **Content lives in data files**: all site content (mod catalog, game modes, launcher features/changelog, radio track list, archived builds, bundles, external links) is hardcoded in `src/data/` (`mods.ts`, `modes.ts`, `launcher.ts`, `radio.ts`, `versions.ts`, `bundles.ts`, `links.ts`). Edit these to update the catalog тАФ pages in `src/pages/` (and the home sections in `src/sections/`) render from them.
- **Assets are optional / drop-in**: the site renders fully with no local image assets (visuals are CSS gradients/SVG). Real art can be dropped into `public/assets/` using the filenames documented in `public/assets/README.md` (`logo.svg`, `hero-art.png`, `og-image.png`, `mods/<mod-id>.png`). Mod cards try a local `/assets/mods/<id>.png` first, then fall back to a remote `raw.githubusercontent.com` thumbnail, then to a generated initials tile тАФ so mod thumbnails depend on outbound network access unless local overrides are added. The committed `og-image.png` is generated from `scripts/og-template.html` via headless Chrome (command documented in `public/assets/README.md`); nothing references the template at runtime.
- **Command palette hotkeys**: `/` and `Ctrl/Cmd+K` globally open the command palette (`src/components/CommandPalette.tsx`, mounted in `Layout`). The listener ignores keystrokes inside `input`/`textarea`/`select`/`[contenteditable]`, so plain form fields are safe тАФ but avoid adding new global `/` key handlers or `preventDefault`-ing `/` in new components.
- **HashRouter vs in-page anchors**: because routing is hash-based, plain `href="#foo"` anchors would be interpreted as a route change (тЖТ 404). The skip link in `src/Layout.tsx` works around this with `preventDefault` + manual focus/scroll тАФ copy that pattern for any new in-page anchor.
- **Reduced-motion gotcha**: the global `prefers-reduced-motion` rule in `src/index.css` zeroes `animation-duration`/`transition-duration` but NOT `transition-delay`/`animation-delay` тАФ components with staggered delays (mobile nav overlay, `useReveal({ stagger: true })`) must zero their own delays via a `matchMedia` check, as the existing ones do.
- **i18n тАФ ALL UI strings live in `src/i18n/en.ts` + `de.ts`**: `en.ts` defines the `Dict` shape and `de.ts` ends with `satisfies Dict`, so `tsc -b` (part of `npm run build`) fails if a key is missing in either file тАФ always update BOTH when adding/removing strings. Site data in `src/data/` (mod catalog, changelog, track namesтАж) stays English by design and must NOT be moved into the dictionaries.
- **Locale persistence**: the active locale is stored in `localStorage` under `'bapbap-locale'` and applied to `<html lang>` (an inline script in `index.html` sets it pre-boot to avoid a language flash; `LanguageProvider` keeps it in sync). When testing via CDP/automation, set the storage key and reload тАФ hash-only navigations do not re-read it.
- **Service worker is PROD-only**: `src/main.tsx` registers `/sw.js` only when `import.meta.env.PROD`, so it exists on `vite preview` (:4173) but never on dev :5173 тАФ dev always serves fresh code with no SW in the way. To test offline flows, build + preview and use CDP Network offline emulation against :4173. When changing any precached file (`index.html`, `favicon.svg`, `manifest.webmanifest`), bump the `CACHE` version in `public/sw.js` so old caches are dropped on activate. Never add remote mod thumbnails (raw.githubusercontent.com) to the SW cache тАФ the initials tile is the designed offline fallback.
- **Generated icons/og-image**: `public/assets/og-image.png`, `icon-512.png`, `icon-192.png` and `apple-touch-icon.png` are GENERATED via headless Chrome from `scripts/og-template.html` / `scripts/icon-template.html` (commands in `public/assets/README.md`). In this VM the default Chrome profile is locked by a running instance тАФ use `--headless=old --no-sandbox --user-data-dir=/tmp/<something>`, and note `--headless=old` may not exit on its own (wrap in `timeout 60 тАж`; the screenshot is written regardless).
- **Preset effect classes are hover-gated by design**: the `.fx-*` classes in `src/index.css` (card-visuals gallery) only animate on `:hover` / `:focus-visible` / `.fx-active` тАФ a perf rule so 28 swatches never animate concurrently while idle. Don't "fix" idle swatches into animating.
